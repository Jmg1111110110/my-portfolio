# 자율 컨텐츠 1 - BOSS(Basic Open Source Software) 프로젝트 소개

## 1. 프로젝트 개요

BOSS(Basic Open Source Software)는 백준(BOJ)이나 Programmers와 같은 온라인 채점 시스템을 직접 구현해보는 팀 프로젝트입니다.  
사용자가 문제를 선택한 뒤 소스코드를 제출하면, 서버에서 코드를 컴파일하고 실행한 후 테스트케이스와 비교하여 채점 결과를 반환하는 방식으로 구성되었습니다.

이 프로젝트는 Java와 Git/GitHub를 활용한 팀 프로젝트 과제로 진행되었으며, 협업 과정과 역할 분담을 경험해볼 수 있는 좋은 기회였습니다.

## 2. 프로젝트 목적

이 프로젝트의 목적은 Java를 활용하여 기본적인 온라인 채점 시스템의 구조를 이해하고,  
Git과 GitHub를 이용한 협업 방식에 익숙해지는 것이었습니다.

또한 단순히 기능을 구현하는 것을 넘어, 팀원들과 함께 역할을 나누고 프로젝트를 완성하는 과정을 직접 경험하는 데 의미가 있었습니다.

## 3. 사용 기술

- HTML
- Java
- Git
- GitHub

## 4. 프로젝트 형태

- 팀 프로젝트

## 5. 내가 맡은 역할

저는 이 프로젝트에서 **백엔드 개발**을 담당하였습니다.  
사용자의 문제 선택과 코드 제출이 자연스럽게 이어질 수 있도록 데이터 흐름을 처리하고, 제출 이후 채점 결과가 화면에 표시될 수 있도록 관련 기능을 구현하였습니다.

## 6. 구현 내용

제가 프로젝트에서 구현한 주요 내용은 다음과 같습니다.

- 문제 선택값 저장 및 제출 페이지 연동 기능 구현
- 코드 제출 처리 기능 개발
- 채점 결과 표시 영역 구성
- 사용자 제출 상태 관리
- 페이지 간 데이터 처리 기능 구현

이 과정을 통해 사용자의 입력이 어떻게 저장되고 전달되는지,  
그리고 하나의 기능이 여러 페이지와 연결되어 동작하기 위해 어떤 처리가 필요한지 배울 수 있었습니다.

## 7. 어려웠던 점

가장 어려웠던 점은 Git과 GitHub를 처음 본격적으로 사용해보는 프로젝트였다는 점입니다.  
초반에는 작업 내용을 반영하고 공유하는 과정이 익숙하지 않아 적응하는 데 시간이 필요했습니다.

특히 협업 환경에서 파일을 관리하고 변경 사항을 정리하는 과정이 처음에는 어렵게 느껴졌지만,  
프로젝트를 진행하면서 점차 사용 방식에 익숙해질 수 있었습니다.

## 8. 배운 점

이 프로젝트를 통해 Git과 GitHub의 필요성을 직접 체감할 수 있었습니다.  
처음에는 낯설고 어렵게 느껴졌지만, 점점 익숙해지면서 팀원들과 작업 내용을 공유하고 소통하는 과정이 훨씬 편리하다는 점을 느꼈습니다.

또한 백엔드 역할을 맡아 데이터 처리와 제출 흐름을 구현해보면서,  
단순한 화면 구성이 아니라 기능이 실제로 동작하도록 만드는 과정에 대해 더 깊이 이해할 수 있었습니다.

## 9. 결과 및 느낀 점

BOSS 프로젝트는 수업 과제로 최종 제출까지 완료하였습니다.  
이 프로젝트는 단순히 과제를 끝내는 것에 그치지 않고, 협업 도구 사용 경험과 백엔드 개발 경험을 함께 쌓을 수 있었던 의미 있는 활동이었습니다.

앞으로도 이러한 경험을 바탕으로, 기능 구현 능력뿐 아니라 협업 능력도 함께 키워나가고 싶습니다.
